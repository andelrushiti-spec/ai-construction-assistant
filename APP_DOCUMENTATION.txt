WHAT IS THIS APPLICATION?
================================================================================

This is a smart assistant application built specifically for construction
companies in Albania. It helps users understand their construction contracts
and Albanian construction laws by letting them ask questions in natural
language (both Albanian and English).

Think of it as having a smart lawyer assistant that has read all your
contracts and construction laws, and can answer your questions about them
instantly.


================================================================================
HOW IT WORKS - IN SIMPLE TERMS
================================================================================

1. YOU UPLOAD YOUR DOCUMENTS
   - Upload your construction contracts (PDF, Word, Pages formats)
   - Upload Albanian construction law documents
   - The system reads these documents and breaks them into small pieces

2. THE SYSTEM PROCESSES DOCUMENTS
   - The application reads every page of your documents
   - It breaks the text into small "chunks" (pieces of about 800 words each)
   - Each chunk gets converted into a special format called an "embedding"
   - These embeddings are like fingerprints for each piece of text
   - All embeddings are stored in a database on YOUR computer

3. YOU ASK QUESTIONS
   - You type a question in Albanian or English
   - Example: "What are the payment terms in my contract?"
   - Example: "What does Albanian law say about concrete safety?"

4. THE SYSTEM FINDS RELEVANT INFORMATION
   - Your question also gets converted into an "embedding"
   - The system compares your question's fingerprint with all the document
     fingerprints
   - It finds the 5 most similar/relevant pieces of text from your documents

5. AI GENERATES AN ANSWER
   - The relevant pieces are sent to OpenAI's GPT-4 (an artificial
     intelligence)
   - GPT-4 reads the relevant pieces and your question
   - It writes a clear, accurate answer in the same language you asked
   - The answer includes citations (which document and page it came from)

6. YOU GET YOUR ANSWER
   - You see the answer with exact citations
   - You can click to see the exact paragraph from the source document
   - You can copy the answer or the source text
   - All your questions and answers are saved in a conversation history

================================================================================
PRIVACY & DATA SECURITY - VERY IMPORTANT
================================================================================

WHERE YOUR DATA IS STORED:

1. LOCAL DATABASE (On Your Computer)
   - User accounts (username, email, encrypted password)
   - Project information
   - Document metadata (filenames, upload dates)
   - Your questions and answers
   - Citations and source references
   - Location: "construction_assistant.db" file on your computer
   - PRIVACY: This data NEVER leaves your computer except for API calls

2. LOCAL FILE STORAGE (On Your Computer)
   - Original uploaded documents (PDF, Word, Pages files)
   - Vector embeddings (numeric representations of your documents)
   - Location: "uploads/" and "vector_db_storage/" folders
   - PRIVACY: These files stay on YOUR computer only

3. WHAT GETS SENT TO OPENAI:

   When Processing Documents:
   - The TEXT from your documents is sent to OpenAI to create embeddings
   - OpenAI converts text to numbers (embeddings) and sends them back
   - According to OpenAI's policy: Data sent to their API is NOT used for
     training their models
   - Your document text is processed and then deleted from their servers

   When Answering Questions:
   - Your question is sent to OpenAI
   - The 5 most relevant chunks from your documents are sent
   - GPT-4 generates an answer and sends it back
   - OpenAI does NOT store your questions or document chunks for training


SECURITY MEASURES:

1. PASSWORD PROTECTION:
   - Your password is NEVER stored as plain text
   - It's "hashed" using PBKDF2 encryption
   - Even the database administrator cannot see your real password
   - Only you can access your projects and documents

2. USER ISOLATION:
   - Each user has their own separate projects
   - You can ONLY see and access YOUR own documents
   - Other users cannot see your projects even if they use the same computer

3. SESSION SECURITY:
   - Login sessions expire after 1 hour of inactivity
   - Sessions use secure tokens
   - HTTPS should be used in production (requires SSL certificate)

4. FILE UPLOAD SECURITY:
   - Only specific file types are allowed (PDF, DOCX, DOC, PAGES)
   - Maximum file size is 50MB
   - Files are scanned and validated before processing

5. API KEY PROTECTION:
   - Your OpenAI API key is stored in environment variables
   - It's not visible in the code or database
   - It's kept in a separate ".env" file


WHAT OPENAI CAN SEE:
- The text content from your uploaded documents (when creating embeddings)
- Your questions
- The relevant chunks sent for answering questions
- They DO NOT see: your username, password, project names, file names

WHAT OPENAI CANNOT DO:
- Store your data for training AI models (per their API policy)
- Access your full documents (only chunks you query about)
- See who you are or your account information
- Share your data with third parties


DATA RETENTION:
- All your data stays on your computer indefinitely until you delete it
- You can delete individual questions/answers
- You can delete entire conversations
- You can delete entire projects (this deletes all related documents and data)
- When you delete, data is PERMANENTLY removed from the database


================================================================================
HOW DATA FLOWS THROUGH THE SYSTEM
================================================================================

DOCUMENT UPLOAD FLOW:
1. You select a PDF/Word document on your computer
2. File is uploaded to the "uploads/" folder on your computer
3. Python reads the document and extracts all text
4. Text is split into chunks (800 words each, 100 word overlap)
5. Each chunk is sent to OpenAI API to create an embedding
6. Embeddings (just numbers) are saved to "vector_db_storage/" on your computer
7. Original file metadata is saved to SQLite database
8. (Optional) Original file can be auto-deleted to save space

QUESTION & ANSWER FLOW:
1. You type a question in the web interface
2. Question is sent to the Flask backend server
3. Question is converted to an embedding using OpenAI API
4. Your computer searches the local vector database for similar embeddings
5. Top 5 most relevant chunks are retrieved from local storage
6. Question + relevant chunks are sent to OpenAI GPT-4
7. GPT-4 generates an answer with citations
8. Answer is saved to your local database
9. Answer is displayed to you with source citations


================================================================================
DATABASE STRUCTURE
================================================================================

The SQLite database contains these tables:

USERS TABLE:
- id: Unique number for each user
- username: Your username (unique)
- email: Your email address (unique)
- password_hash: Your encrypted password
- created_at: When you created your account

PROJECTS TABLE:
- id: Unique number for each project
- name: Project name (like "Tirana Construction Project")
- description: Optional description
- user_id: Links to the user who owns it
- contract_index_path: Where contract embeddings are stored
- law_index_path: Where law embeddings are stored
- created_at: When project was created
- updated_at: When project was last modified

CONTRACTS TABLE:
- id: Unique number for each contract
- project_id: Which project it belongs to
- filename: Stored filename
- original_filename: Original filename you uploaded
- file_size: Size in bytes
- pages: Number of pages
- uploaded_at: When you uploaded it
- processed: Whether it's been converted to embeddings
- processing_status: pending/processing/completed/failed

LAWS TABLE:
- Same structure as contracts
- Plus: law_number, law_title, year, update_version

CONVERSATIONS TABLE:
- id: Unique number for each conversation
- project_id: Which project it belongs to
- user_id: Who started it
- title: Auto-generated from first question
- created_at: When conversation started
- updated_at: Last message time

QUERIES TABLE (messages in conversations):
- id: Unique number for each message
- conversation_id: Which conversation it belongs to
- project_id: Which project
- user_id: Who asked
- question: Your question text
- answer: AI's answer text
- language: 'sq' (Albanian) or 'en' (English)
- contract_citations: JSON with source references from contracts
- law_citations: JSON with source references from laws
- response_time: How long it took to answer (in seconds)
- created_at: When you asked the question

TECHNICAL ARCHITECTURE
================================================================================

FRONTEND (What you see):
- HTML pages with forms and display areas
- CSS styling for professional appearance
- JavaScript for interactivity and API calls
- Runs in your web browser

BACKEND (Server):
- Flask web server handles requests
- Python processes documents and searches
- Routes (endpoints) for different functions:
  * /api/auth/register - Create account
  * /api/auth/login - Log in
  * /api/projects - Manage projects
  * /api/upload/contract - Upload contracts
  * /api/upload/law - Upload laws
  * /api/query - Ask questions
  * /api/conversations - Manage conversations

DOCUMENT PROCESSING PIPELINE:
1. File Upload Handler
2. Text Extractor (PDF/Word reader)
3. Text Chunker (splits into 800-word pieces)
4. Embedding Generator (calls OpenAI API)
5. Vector Database (stores embeddings locally)

QUERY PROCESSING PIPELINE:
1. Question Receiver
2. Language Detector
3. Embedding Generator (convert question to embedding)
4. Vector Search (find similar chunks in database)
5. Context Builder (prepare chunks for GPT-4)
6. LLM Query (send to GPT-4)
7. Response Formatter (format answer with citations)
8. Database Saver (save Q&A to database)

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

COMMON ISSUES:

1. "OPENAI_API_KEY not found"
   - Create a .env file in the root directory
   - Add: OPENAI_API_KEY=your-api-key-here

2. "Database locked" error
   - Close all instances of the application
   - Restart the server

3. Documents not processing
   - Check file format (must be PDF, DOCX, DOC, or PAGES)
   - Check file size (must be under 50MB)
   - Check logs/app.log for error details

4. Can't login
   - Clear browser cookies
   - Try different browser
   - Check if username/password is correct

5. Slow responses
   - Normal for first question (loading embeddings)
   - Check internet connection speed
   - OpenAI API might be slow during peak times
